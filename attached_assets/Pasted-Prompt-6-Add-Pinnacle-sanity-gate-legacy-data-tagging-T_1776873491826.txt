CHANGE A — Pinnacle sanity gate

Semantics check: positive edge means actionable < fair (exchange offering better than fair — good). Negative edge beyond -3pp means we'd be taking a price that's significantly worse than sharps think fair — the "too good to be true from the wrong direction" scenario, or a sanity signal that something's off with the pricing.

Alternative framing: reject if actionablePrice is more than X above fairValueOdds by enough that the implied-prob gap exceeds 3pp. Use implied-prob difference for the check, not raw odds ratio.

Actually wait — the direction depends on what we're trying to catch. Reconsider:

Both directions of large gap are suspicious. The sanity gate should reject when the absolute gap is too large.

Corrected implementation:

Wait — that rejects ALL large edges, which defeats the purpose.

Think about this differently. The purpose is to catch "too good to be true" offerings, which means exchange price WAY better than sharps think reasonable. That's fairValueImplied < actionableImplied by a large amount (sharps say "unlikely", exchange says "even more unlikely"). The CLV-style edge formula is (fairValueImplied − actionableImplied). If this is very positive, it's "too good" — reject.
